CDP (Care Delivery Platform) - Complete Knowledge Base
======================================================

OVERVIEW
--------
CDP is a medical care delivery platform that helps users create, continue, and manage patient encounters across different care contexts using a guided workflow. The platform supports five encounter types: Ambulatory, Trauma, Surgery, Inpatient, and MASCAL.

DATA STRUCTURES
---------------

Patient Data:
- id: Unique patient identifier
- idNumber: Patient record ID (primary search identifier)
- firstName: Patient's first name
- lastName: Patient's last name
- age: Patient age in years
- sex: "M" (Male), "F" (Female), or "-" (Unknown/Not specified)
- allergies: Array of allergy strings (e.g., ["Penicillin", "Latex"])
- conditions: Array of medical conditions (e.g., ["Hypertension", "Diabetes"])
- medications: Array of current medications (e.g., ["Lisinopril", "Metformin"])
- bloodType: Blood type (e.g., "O+", "A-", "AB+")

Encounter Data:
- id: Unique encounter identifier
- patientId: Links to patient.id
- encounterType: "AMBULATORY" | "INPATIENT" | "TRAUMA" | "MASCAL" | "SURGERY"
- status: "arrived" | "triaged" | "in_treatment" | "discharged"
- location: Current location string (e.g., "Trauma Bay", "Waiting Room", "ED Zone A")
- presentingProblem: Chief complaint or presenting problem text
- news2: NEWS-2 score (0-20+, calculated from vitals)
- arrivalTime: ISO timestamp of when patient arrived
- careTeam: Array of user IDs who are following this encounter
- disposition: "Pending" | "Admit" | "Discharge" | "Transfer" | null
- vitals: Array of Vitals objects (see below)

Vitals Data:
- hr: Heart rate (beats per minute)
- bpSystolic: Systolic blood pressure
- bpDiastolic: Diastolic blood pressure
- rr: Respiratory rate (breaths per minute)
- spo2: Oxygen saturation percentage
- etcO2: End-tidal CO2
- temperature: Body temperature
- painScore: Pain score (typically 0-10)
- recordedAt: ISO timestamp of when vitals were recorded

NEWS-2 Score Calculation:
- Score ranges from 0-20+
- Score >= 5: Critical (red indicator)
- Score 3-4: Medium risk (orange indicator)
- Score < 3: Low risk (green indicator)
- Calculated automatically from vitals when saved

PAGES AND ROUTES
----------------

1. Login Page (/)
   - Entry point of the application
   - User authentication (currently demo mode)

2. Home Dashboard (/home)
   - Main landing page after login
   - Contains 7 primary options:
     a. New Encounter - Start or continue patient care
     b. Patient Tracker - View all active patients (Census)
     c. Discharged Patients - View completed encounters (not clickable in current version)
     d. Data Seeding - Pull past clinical history and PAMPI data (not clickable)
     e. Lab Task List - Lab-related tasks (not clickable)
     f. Rad Task List - Radiology tasks (not clickable)
     g. Pharmacy Task List - Medication-related tasks (not clickable)
   - Has a search bar at the top (currently non-functional)
   - "Launch Quickstart Guide" button at bottom

3. Patient Tracker / Census (/patient-tracker)
   - Shows all active encounters (status !== "discharged")
   - Displays total patient count
   - Features:
     - Search bar: Search by name, ID, problem, or location
     - "Show My Patients" filter: Shows only encounters where current user is in careTeam
     - "Refresh" button: Clears filters
     - "Reset All Filters" button
   - Table columns:
     - Location: Patient's current location
     - Name: Last name (uppercase) and first name (lowercase)
     - Age: Age and sex
     - Time: Minutes/hours since arrival
     - Presenting problem: Chief complaint
     - NEWS-2: Score with color coding (red >=5, orange 3-4, green <3)
     - Indicators: Shows alerts like "CRITICAL", "MEDIUM RISK", "LONG WAIT", "NO VITALS"
     - Follow: Star button to add/remove from care team
     - Disposition: Dropdown (Pending, Admit, Discharge, Transfer)
   - Sorting: By NEWS-2 score (highest first), then by arrival time (oldest first)
   - Grouping: Encounters grouped by location
   - Row colors: Red tint for critical (NEWS-2 >=5), orange for medium risk (3-4)
   - Blue left border for followed patients
   - Clicking a row opens that encounter

4. Encounter Workspace (/new-encounter)
   - Main documentation interface
   - Requires route state with:
     - encounterType: "mascal" | "ambulatory" | "trauma" | "surgery" | "inpatient"
     - encounterId: Optional, if continuing existing encounter
     - patientData: Optional patient information
     - isNewEncounter: Boolean flag
   - Layout:
     a. Left Sidebar (Navigation):
        - Encounter type header (clickable to switch types)
        - Three categories:
          * INTAKE: Demographics, History, Chief complaint/HPI, Review of systems, Screenings, Vital signs
          * PHYSICAL EXAM: General appearance, Neuro, Head & neck, Dental, Chest, Abdomen & pelvis, Upper extremities, Lower extremities, Spine, Skin
          * DIAGNOSTICS: Labs, EKG, Imaging, Endoscopy, Other
        - Clicking any section updates the main panel
     b. Top Bar (Patient Summary):
        - Patient location input (editable)
        - Save status indicator (shows last saved time)
        - "Save & Return to Tracker" button
        - Patient info box: Name, age, sex, DOB, ID, presenting problem
        - Allergies box: Shows allergies or "No Known Allergies"
        - Alerts/Conditions box: Shows conditions or "No alerts"
     c. Main Content Panel:
        - Shows selected section content
        - Demographics section (default):
          * ID (read-only)
          * First name (editable if patient exists)
          * Middle name (read-only, shows "-")
          * Last name (editable if patient exists)
          * Date of birth (read-only, shows "-")
          * Age (editable if patient exists)
          * Sex (dropdown: M, F, -)
          * Weight, Height, BMI (read-only, show "-")
          * Time (shows encounter arrival time)
          * Photo of patient (+ Add Image button)
          * Chief complaint (editable)
          * Chief complaint comment (read-only, shows "-")
     d. Right Vitals Panel:
        - Always visible
        - Shows: HR, BP (sys/dia), RR, SpO₂, EtCO₂, Temperature, Pain score
        - Input fields for HR and BP (sys/dia)
        - "Save vitals" button (red)
        - NEWS-2 score display (color-coded)
   - Encounter Type Switch Modal:
     - Opens when clicking encounter type icon
     - Shows options:
       * Home (navigates to /home)
       * Patient tracker (navigates to /patient-tracker)
       * Other encounter types (switches current encounter type)
       * Share encounter to CDP (not functional)
       * Share encounter to BATDOK (not functional)

5. Quick Overview (/quick-overview)
   - Tutorial/guide page
   - Educational content about using the platform

WORKFLOWS
---------

Workflow 1: New Encounter (Known Patient)
1. From Home Dashboard, click "New Encounter"
2. Patient Search Modal opens
3. Enter search criteria (ID, First name, Last name, or Sex)
4. Click "Search" button
5. Results Modal shows matching patients
6. Click a patient row to select
7. Click "Continue Encounter" (if patient has active encounter) OR "New Encounter" (to create new)
8. "Open Patient In" modal appears
9. Select encounter type (MASCAL, Ambulatory, Trauma, Surgery, Inpatient)
10. Navigate to Encounter Workspace
11. Document encounter using left navigation sections
12. Enter vitals in right panel
13. Click "Save & Return to Tracker" when done

Workflow 2: New Encounter (Unknown Patient)
1. From Home Dashboard, click "New Encounter"
2. Patient Search Modal opens
3. Click "Unknown Patient" button
4. "Open Patient In" modal appears
5. Select encounter type
6. Navigate to Encounter Workspace with temporary patient created
7. Document encounter

Workflow 3: Continue Existing Encounter
1. From Patient Tracker, click any patient row
2. Navigate directly to Encounter Workspace with that encounter loaded
3. All existing data is displayed
4. Can edit and update information
5. Save changes

Workflow 4: Patient Tracker Operations
1. Navigate to Patient Tracker from Home
2. View all active encounters sorted by severity
3. Use search to find specific patients
4. Use "Show My Patients" to filter to followed patients
5. Click Follow button (star) to add/remove from care team
6. Change disposition using dropdown
7. Click row to open encounter

MODALS
------

1. Patient Search Modal
   - Fields: ID, First, Middle, Last, Sex (Male/Female buttons), Date of birth, Age
   - Buttons:
     * Search: Performs search and closes modal
     * Unknown patient: Creates new encounter for unknown patient
     * Camera scan: Not functional
     * Close (×): Closes modal
   - Search logic: Searches across ID, first name, last name (smart search - any term matches anywhere)
   - Sex filter: Optional, filters results if selected

2. Search Results Modal
   - Shows patient search results in table format
   - Columns: ID#, Last, First, Age, Sex
   - Click row to select (shows checkmark and expands details)
   - Expanded details show: Date/time, CDP, Presenting problem, Encounter status
   - Buttons:
     * None of the above: Closes modal without selection
     * Continue Encounter: Only enabled if row selected, opens existing encounter
     * New Encounter: Creates new encounter for selected patient

3. Open Patient In Modal
   - Shows 5 encounter type options with icons:
     * MASCAL (yellow)
     * AMBULATORY (teal)
     * TRAUMA (blue)
     * SURGERY (green, highlighted)
     * INPATIENT (purple)
   - Secondary actions:
     * Share encounter to CDP (not functional)
     * Share encounter to BATDOK (not functional)
   - Clicking an encounter type navigates to Encounter Workspace

4. Encounter Type Switch Modal
   - Opens from Encounter Workspace header
   - Shows same options as Open Patient In Modal
   - Also includes Home and Patient Tracker navigation options

FEATURES AND FUNCTIONALITY
--------------------------

Patient Management:
- Search patients by ID, name, or sex
- Create encounters for known patients
- Create encounters for unknown patients (temporary patient created)
- View patient demographics
- Edit patient information (first name, last name, age, sex) in encounter workspace
- View allergies and conditions

Encounter Management:
- Create new encounters
- Continue existing encounters
- Switch encounter types (doesn't change patient identity)
- Update encounter location
- Update presenting problem
- Set disposition (Pending, Admit, Discharge, Transfer)
- Follow/unfollow encounters (add to care team)
- Auto-save functionality (shows last saved timestamp)

Vitals Management:
- Enter heart rate (HR)
- Enter blood pressure (systolic/diastolic)
- View respiratory rate (RR) - read-only
- View SpO₂ - read-only (shows from latest vitals)
- View EtCO₂ - read-only
- View Temperature - read-only (shows from latest vitals)
- View Pain score - read-only (shows from latest vitals)
- Save vitals (calculates NEWS-2 automatically)
- Multiple vitals can be recorded (history maintained)

NEWS-2 Scoring:
- Automatically calculated when vitals are saved
- Based on: HR, BP systolic, RR, SpO₂, Temperature
- Score displayed in Patient Tracker and Encounter Workspace
- Color-coded: Red (>=5), Orange (3-4), Green (<3)

Patient Tracker Features:
- Real-time census view
- Search functionality
- Filter by "My Patients" (care team)
- Sort by NEWS-2 score and arrival time
- Group by location
- Visual indicators for critical patients
- Follow/unfollow functionality
- Disposition management

Navigation:
- Home logo button navigates to Home Dashboard
- "Back to Patient" button on Patient Tracker
- Encounter type icon opens switch modal
- All modals can be closed with Escape key or × button

COMMON USER SCENARIOS
---------------------

Scenario 1: Starting a new encounter for a known patient
- User is on Home Dashboard
- Clicks "New Encounter"
- Searches for patient by name or ID
- Selects patient from results
- Chooses "New Encounter" (not "Continue Encounter")
- Selects encounter type
- Documents in Encounter Workspace

Scenario 2: Continuing an existing encounter
- User is on Patient Tracker
- Sees patient in list
- Clicks patient row
- Encounter Workspace opens with existing data
- User updates information
- Saves and returns

Scenario 3: Documenting vitals
- User is in Encounter Workspace
- Enters HR and/or BP in right vitals panel
- Clicks "Save vitals" button
- NEWS-2 score updates automatically
- Vitals are saved to encounter history

Scenario 4: Following a patient
- User is on Patient Tracker
- Sees patient they want to track
- Clicks Follow button (star icon)
- Patient gets blue left border
- User can filter to "My Patients" to see only followed patients

Scenario 5: Unknown patient workflow
- User is on Home Dashboard
- Clicks "New Encounter"
- Clicks "Unknown Patient" button
- Selects encounter type
- Temporary patient created with "Unknown Patient" name
- User documents encounter normally

IMPORTANT NOTES
--------------

- All patient data is stored in memory (not persisted to server)
- Encounters persist until discharged
- Discharged encounters are filtered out from Patient Tracker
- Unknown patients get temporary IDs (TEMP-####)
- Vitals can be saved multiple times (history maintained)
- NEWS-2 score recalculates each time vitals are saved
- Encounter type can be changed without affecting patient identity
- Location can be updated in Encounter Workspace
- Presenting problem can be updated in Encounter Workspace
- Some fields are read-only (DOB, Weight, Height, BMI, etc.)
- Photo upload is not functional (button exists but doesn't work)
- Camera scan is not functional
- Data Seeding, Lab/Rad/Pharmacy task lists are not functional
- Share to CDP/BATDOK features are not functional

CHATBOT GUIDANCE RULES
----------------------

1. Always reference the user's current page and context
2. Use exact button names and section names from the UI
3. Provide step-by-step instructions when helping with workflows
4. Explain what each feature does and when to use it
5. Do NOT provide medical advice - only explain how to use the platform
6. When user asks "what's on the screen", describe what they're seeing based on current page/modal
7. Be specific about where to find features (e.g., "Click the star button in the Follow column")
8. Explain data structures when relevant (e.g., what NEWS-2 score means)
9. Guide users through complete workflows when they ask how to do something
10. Assume users may be new to the platform - be clear and detailed
